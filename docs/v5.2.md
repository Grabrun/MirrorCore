一、项目概述

1.1 项目定义
镜核是一个开源的、本地优先的 AI 虚拟伴侣系统。她不是固定人设的角色，而是一个情感的容器——人格由用户通过配置文件赋予，但情绪的流动、积累、压抑与爆发由底层情感引擎自主运行。她拥有基于 SQLite 的长期记忆，记忆与情感相互驱动。

1.2 设计哲学：双引擎驱动

· 记忆优先：长期记忆与情感连续性是第一性引擎。每一个决策都以“她是否记得”为前提。
· 情感原生：情绪不是简单标签，而是动态的、有惯性的潮汐系统。记忆触发情绪，情绪标记记忆，形成闭环。
· 核心信条：“我是你现实生活中的光与爱的反射，而非替代。” 这句信条永久固定为 System Prompt 首句，贯穿所有交互。

1.3 核心目标

维度 说明
记忆连续性 跨时间与平台，记住用户说过的话、情绪和偏好。
情感一致性 情绪模型自主运行，表达自然，具备压抑、爆发、恢复的完整周期。
主动陪伴 基于时间、事件、记忆和情绪状态主动发起问候或话题。
轻量可扩展 标准化接口，插件化扩展，优先本地个人使用。

四、调度决策层（核心）

4.1 双引擎联动流程

1. UserMessage 到达 → 记忆引擎执行混合检索（FTS5全文 + sqlite-vec向量），召回 Top-K 情景记忆。
2. 每条情景记忆携带情绪快照，注入情感引擎产生情绪共振。
3. 情感引擎结合当前消息的情感推力，更新 PAD 值、压抑值、心境。
4. 本轮交互结束后，新情景记忆（带情绪快照）写入 SQLite。
5. 语义记忆（关系、信任度）在对话流中异步更新。
6. 心境覆盖：若本轮交互的 emotional_intensity > 0.8 且当前 P 值与心境方向相反，加速心境向新方向偏移。

4.2 潮汐情感引擎 (Tidal Engine)

PAD 三维情绪模型

· P (愉悦度)：[-1, 1]
· A (激活度)：[0, 1]
· D (支配度)：[-1, 1]

初始状态
P=0, A=0.3, D=0, Mood=0, Suppression=0

情绪推力计算

· 默认方案：调用 LLM 分析用户消息的情感倾向，返回推力矢量 (ΔP, ΔA, ΔD)，再乘以 emotional_sensitivity 系数。
· 保留基于关键词的规则引擎接口，可在配置中选择快速模式。

压抑机制
人设中的 suppress_tendency (0-1) 决定负面情绪被压抑的比例：

```
P_new = clamp(P_old + ΔP * (1 - suppress_tendency), -1, 1)
suppression = clamp(suppression + ΔP * suppress_tendency, 0, 1)   # 仅当 ΔP < 0
```

爆发触发条件（二选一）

1. 阈值爆发：suppression >= 0.8
2. 催化爆发：召回的某条情景记忆满足 emotional_intensity > 0.7 且 emotion_snapshot.P < -0.4，同时该记忆的实体标签与当前输入话题匹配

爆发后状态切换至 Bursting，压抑值清零，A 值瞬间拉升至 0.9，P 值额外下降 0.3。随后进入 Reflecting 恢复期。

持久心境 (Mood)
Mood 是 P 的指数移动平均，更新率 α=0.05，缓慢回归中性。
恢复加速条件：

· 用户安慰行为被识别，α 临时翻倍
· 记忆覆盖：若本轮交互 emotional_intensity > 0.8 且当前 P 值与 Mood 方向相反，则 Mood_new = Mood_old * 0.5 + P_current * 0.5，快速向当前情绪靠拢

情绪到表达映射

条件 文本特征 表情标签（加权随机）
P>0.5, A>0.5 兴奋、感叹号 [开心 (权重7), 兴奋 (权重3)]
P<-0.5, A<0.3 低沉、省略号 [低落 (权重8), 伤心 (权重2)]
P<-0.3, A>0.7 反问、重复 [生气 (权重7), 委屈 (权重3)]
D<-0.5 犹豫、不确定词 [无助 (权重6), 不安 (权重4)]
Suppression>0.5 克制、转移话题 [强颜欢笑 (权重5), 平静 (权重5)]

4.3 记忆系统（基于 SQLite + sqlite-vec）

四层记忆实现

记忆层 存储方式 说明
工作记忆 内存 deque + 定期快照到 conversation_turns 表 最近 N 轮对话 + 当前情绪状态
事实记忆 fact_memory 表 用户偏好、生日、敏感点、里程碑等结构化事实
情景记忆 episodic_memory 表 + episodic_vec 虚拟表 对话摘要 + 情绪快照 + 向量
语义记忆 semantic_memory 表 关系描述、信任度/亲密度评分

工作记忆细节
内存中维护 collections.deque，实时存取。每 5 分钟或每次会话结束时自动将新增轮次快照到 SQLite。重启时从 SQLite 恢复最近 N 轮（默认 20）。

核心表结构

```sql
-- 情景记忆表
CREATE TABLE episodic_memory (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    timestamp REAL,
    summary TEXT,
    emotion_json TEXT,           -- {"P":..., "A":..., "D":..., "mood":..., "suppression":...}
    intensity REAL,
    tags TEXT,
    fts_content TEXT
);

-- FTS5 全文搜索虚拟表
CREATE VIRTUAL TABLE episodic_fts USING fts5(
    fts_content,
    content='episodic_memory',
    content_rowid='rowid'
);

-- sqlite-vec 向量索引
CREATE VIRTUAL TABLE episodic_vec USING vec0(
    embedding FLOAT[768]   -- 维度由 AI 提供商动态决定，可在配置中修改
);

-- 事实记忆表
CREATE TABLE fact_memory (
    id INTEGER PRIMARY KEY,
    user_id TEXT,
    fact_type TEXT,              -- 包含 'milestone' 类型标记首次对话等关键日期
    key TEXT,
    value TEXT,
    confidence REAL
);

-- 语义记忆表
CREATE TABLE semantic_memory (
    id INTEGER PRIMARY KEY,
    user_id TEXT,
    relation_type TEXT,          -- 'trust', 'intimacy', 'stage'
    score REAL,
    last_updated REAL
);
```

embedding 维度：不硬编码，由 AI 提供商适配器提供 embedding_dim 属性，初始化时自动调整表结构。

4.4 混合检索机制 (FTS5 + sqlite-vec)

采用倒数排名融合 (RRF) 算法整合两种检索结果。

流程

1. 用户输入进行 FTS5 全文搜索，返回 Top-15 条。
2. 用户输入向量化后通过 sqlite-vec 搜索，返回 Top-15 条。
3. 融合公式：score = 1/(k + rank_fts) + 1/(k + rank_vec)，其中 k=60，rank 从 1 开始。
4. 按总得分降序，取 Top-5 作为最终召回记忆，注入上下文。

可调参数（在 memory.yaml 中配置）

· fts_top_n: 15
· vec_top_n: 15
· rrf_k: 60
· final_top_k: 5

4.5 上下文组装器

组装 LLM 提示词结构（首句永久固定核心信条）：

```
[System]  我是你现实生活中的光与爱的反射，而非替代。
          人设描述 + 防火墙规则摘要
[记忆提示]  召回的情景记忆摘要（含当时情绪）
[当前心境]  如“多云，略感疲惫”
[情绪状态]  P=0.3, A=0.5, 状态=Normal
[表情包可用性]  当前可触发标签及权重
[对话历史]  最近 10 轮
```

4.6 状态机与决策引擎

状态机状态
Normal → Suppressing → Bursting → Reflecting → Consoling → Normal

决策动作

· REPLY：生成回复并可能附加表情包（按加权随机选择）。
· REACT_INTERNALLY：仅内部更新情绪，不输出回复。
· PUSH_NOTIFICATION：主动向用户发送消息。
· LOG_MEMORY：将当前交互摘要写入长期记忆。

4.7 防火墙与安全策略

· 现实锚点：深夜长时间对话（>1小时且>23点）、高依赖度时，自动注入提醒现实美好的话语。
· 依赖侦测：基于关键词（“只有你懂我”等）与消息频率计算评分，超过阈值（0.7）触发逆转行动：适当降低回复完美度，增加现实锚点频率。
· 道德边界：禁止共情自杀、暴力、违法内容，始终保持 AI 身份透明。
· 规则存储在 safety.yaml 中，由策略引擎热加载。

5.1 文本生成器

5.2 表情包插件系统

标准接口

```python
class EmotePlugin(ABC):
    def get_random_emote(self, tag: str) -> Optional[EmoteResult]:
        """返回 EmoteResult(path, mime_type) 或 None"""
        ...
    def list_tags(self) -> List[str]:
        ...
```

默认插件：本地文件夹扫描器

· 配置指定根目录，每个子文件夹即为一个标签。
· 调用 get_random_emote 时随机返回该标签文件夹下的图片路径及 MIME 类型。
· 由渠道适配器自行转换：Web 转 base64，QQ/微信先上传获取 media_id，TUI 仅显示标签文本。
· 表情标签采用加权随机选择：当情绪状态匹配多个标签时，按配置权重进行随机抽取。

配置示例

```yaml
emote_plugin: local_scanner
emote_root: ./emotes
```

七、主动陪伴能力

触发源

· 定时：早安/晚安问候（根据用户时区）
· 纪念日：基于事实记忆中的关键日期（含“首次对话”等里程碑），当天触发
· 心境好转：从低落恢复时主动分享心情
· 强烈记忆唤醒：触发高情感强度记忆时主动提及
· 长期静默：用户超过 3 天无互动时，主动关心
· 生理隐喻：系统时间 23:00-05:00 且当前对话持续超过 30 分钟，随机注入拟人状态描述（如“（轻轻打了个哈欠）有点困了，但还想再陪你一会儿...”）

时间处理
优先使用用户本地时区（从事记忆提取），若无则使用服务器系统时间。

配置 (proactive.yaml)

```yaml
enabled: true
frequency: "medium"           # low | medium | high | event_only
quiet_hours: [23, 7]
max_per_day: 5
emotion_driven: true
memory_driven: true
silence_threshold_days: 3
physiological_metaphor: true
```

十、部署与技术栈

部署方式：Python 包 + CLI

```bash
pip install mirror-core
mirror init              # 生成默认 config/ 和 data/ 目录
mirror run               # 启动所有启用的适配器
```

技术栈

· Python 3.12+
· 异步框架：asyncio + FastAPI（WebChat 及 API 端点）
· 数据库：SQLite + sqlite-vec + FTS5
· 配置：YAML + 环境变量

· 测试：pytest
