镜核系统设计补丁 v1.1 (关键决策落地)

变更 1: 用户时区数据来源 (影响: 基础设施层、配置管理、记忆引擎)

· 配置优先级 (P0): persona.yaml 新增字段 timezone: "Asia/Shanghai"，作为主要来源。
· 兜底机制 (P1): 新增 API POST /api/v1/user/timezone，WebChat 前端在初始化时调用上报。微信/QQ 适配器暂不上报。
· 存储策略: 以 fact_type='preference', key='timezone' 格式存入 fact_memory 表，运行时可缓存在内存 UserProfile 对象中。
· 默认行为: 上述来源均无效时，使用服务器系统时区，并记录 warn 日志。

变更 2: quiet_hours 静音时段配置 (影响: 配置管理、定时触发器、主动陪伴模块)

· 配置文件: 新增 proactive.yaml 配置项。
  ```yaml
  quiet_hours:
    enabled: true
    start: 22
    end: 7
  ```
· 语义: 区间为 [start, end) 闭开区间，跨日有效。
· 例外规则: 静音时段内阻止所有非紧急主动消息，但用户主动消息、纪念日追加、心境大幅回升（mood < -0.5 转正）的场景除外。

变更 3: 生理隐喻具体化 (影响: 配置管理、主动陪伴模块、文本生成器)

· 配置文件: proactive.yaml 中新增投放概率与词库。
  ```yaml
  physiological_metaphor:
    enabled: true
    base_probability: 0.25
    max_per_night: 2
    require_active_session: true
  physiological_metaphor_library:
    # ... 见架构师决策中的详细词库
  ```
· 检查与投放逻辑:
  1. ProactiveManager 检查时区时间在 23:00-05:00、有活跃会话、非爆发状态。
  2. 随机数命中后，DecisionEngine 从匹配情感（neutral_tired/low_mood/high_mood）的词库中选取文本，作为 ProactiveChance 事件参数发布。

变更 4: WebChat 匿名认证 (影响: 接入适配层、WebChat前端、基础设施层)

· 认证模式: 轻量级 JWT Token 方案。
· 配置文件: 新增 webchat.yaml。
  ```yaml
  auth:
    mode: "token"
    token_expire_days: 7
    secret_file: "data/.jwt_secret"
  cors:
    allowed_origins: ["http://127.0.0.1:8080", "http://localhost:8080"]
  ```
· API 接口: 新增 POST /api/v1/auth/anonymous，返回 user_id 和 token。
· WebSocket 认证: /ws 端点改为 /ws?token=mr_xxx，服务端在握手时验证 JWT。
· 降级: mode: "none" 时保留原有直传 user_id 逻辑，仅供本地开发。
